# Как вносить изменения

Спасибо за интерес к проекту. Здесь описано, как устроен сайт и как безопасно править контент.

## Структура

Проект — **один самодостаточный файл** `index.html` (HTML + CSS + ванильный JS, без сборки и зависимостей, кроме шрифтов Google Fonts с CDN). Весь контент живёт внутри него в виде JS-объектов:

| Объект в `index.html` | Что хранит |
|------------------------|-----------|
| `var D = {…}` | drill-down по каждому инструменту: список «что нужно знать» (`a`) и ресурсы (`r`) |
| `var GRADE = {…}` | минимальный грейд каждого инструмента: `1` = Junior, `2` = Middle, `3` = Senior |
| блоки `<div class="srow" …>` | сами шаги и чипы-инструменты в правой колонке |
| `<div class="pcard" …>` | 6 столпов компетенций (левая карта) |

Ключ записи в `D` и `GRADE` — это **точный текст чипа** (то, что написано на «таблетке» инструмента).

## Правило по ресурсам (важно)

В ссылках разрешены только **бесплатные** материалы:

- ✅ официальная документация, GitHub-репозитории, статьи в открытом доступе, **бесплатные** курсы (Hugging Face, DeepLearning.AI short courses, LangChain Academy, Pinecone Learn);
- ❌ **никаких книг и платных курсов** (O'Reilly, Coursera-сертификаты, Udemy, Manning и т.п.).

Каждый ресурс — это массив `["ТИП", "Название", "https://…"]`. Допустимые типы: `DOCS`, `GUIDE`, `COURSE`, `REPO`, `TOOL`, `PAPER`.

### Пример: добавить ресурс к инструменту

Найди нужный ключ в `D` и допиши элемент в массив `r`:

```js
"Reranking":{a:[
  "Reranker (модель/API) пересортировывает топ-k по релевантности",
  "Cross-encoder — один из вариантов реализации"
],r:[
  ["DOCS","Cohere — Rerank","https://docs.cohere.com/docs/rerank-overview"],
  ["REPO","BAAI bge-reranker","https://github.com/FlagOpen/FlagEmbedding"]   // ← новый
]},
```

### Пример: изменить грейд инструмента

В объекте `GRADE` поправь число (`1`/`2`/`3`):

```js
"MCP":2,   // было middle → если считаешь senior, поставь 3
```

## Проверка ссылок (обязательно перед PR)

В репозитории есть готовый чек. Прогоняй его после любых правок ссылок:

```bash
python3 check-links.py index.html
```

Скрипт классифицирует каждую ссылку: `OK / REDIRECT / BLOCKED / DEAD / UNREACHABLE`.
**Любая `DEAD` (404/410) должна быть исправлена** — на неё скрипт возвращает ненулевой код выхода (удобно для CI). Редиректы (200 после перехода) допустимы.

## Definition of Done для контентной правки

- [ ] ключи в `D` / `GRADE` совпадают с текстом чипов;
- [ ] добавлены только бесплатные ресурсы (без книг и платных курсов);
- [ ] `python3 check-links.py index.html` — без `DEAD`;
- [ ] сайт открывается локально и интерактив работает (фильтр грейдов, drill-down, прогресс, тёмная тема).

## Локальный запуск

```bash
python3 -m http.server 8000   # → http://localhost:8000
```


## Поддержание актуальности

Экосистема AI меняется быстро. Раз в квартал имеет смысл пересверять модели, ссылки и разметку грейдов — что было «свежим», за полгода устаревает. Прогон `check-links.py` ловит ссылки, отвалившиеся со временем.

---

> Контент изначально собирался скриптом-билдером из отдельных источников (`template + data → index.html`). Если правок много и удобнее вести их через билдер, а не руками в `index.html`, — заведи issue, выложим пайплайн сборки.
